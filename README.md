# Hobby catalog 
# Описание
Хобби каталог настольных игр, где зарегистрированный пользователь может просматривать товары и 
добавлять себе в корзину понравившиеся игры/удалять их и также "покупать" выбранные товары на деньги закинутые на его счет
В каталоге также есть админ, который может работать с товарами непосредственно (добавлять/изменять/удалять товары и их типы)

# Основные ендпоинты
Основные действия с товаром:
1. GET /hobby - показать все товары в каталоге
2. GET /hobby/{id} - показать товар по айди
3. POST /hobby/add - добавить новый товар (для админа)
4. PUT /hobby/{id} - изменить существующий товар (для админа)
5. DELETE /hobby/{id| - удалить товар (для админа)

Основные действия с типом товара (только для админа):
1. GET /type - показать все типы товаров
2. GET /type/{id} - показать тип товара по айди
3. POST /type/add - добавить новый тип товара
4. PUT /type/{id} - изменить существующий тип товара
5. DELETE /type/{id| - удалить тип товара
    
Работа с аккаунтом:
1. POST /auth/login - вход существующего пользователя
2. POST /auth/registration - регистрация нового пользователя (один имейл)
3. POST /auth/refresh - обновление токена

Для зарегистрированного пользователя
1. GET /cart - просмотр товаров корзины
2. POST /cart/{id} - добавление товара в корзину по айди
3. POST/cart/buy/{id} - купить товар
4. POST /cart/wallet - пополнить электронный кошелек
5. DEL /cart/{id} - удаление товара из корзины

# Дополнительный функционал

Можно осуществлять фильтрацию товаров для показа по следующим критериям:
- название типа товара
- минимальное количество игроков
- максимальное количество игроков
- минимальная цена
- максимальная цена
-  По цене (ASC/DESC)
- жанр
* Пагинация: page, size параметры.

Регистрация кошелька:
- номер карты
- год действия
- имя держателя

При "покупке" товара необходимо указать:
- Адрес проживания (улица, дом, индекс почтового ящика). 
Адрес проживания указывается если его нет в таблице. Если он есть и адрес все равно введен 
необходимо обновить данные в таблице
- CVV/CVC карты

# Стек
1. Java 17+ – современная версия языка.
2. Spring Boot – быстрый старт и конфигурация.
3. Spring MVC – реализация REST API.
4. Hibernate + Spring Data JPA – ORM для работы с БД.
5. PostgreSQL – реляционная база данных.
6. JUnit 5 – модульное тестирование.
7. Swagger (OpenAPI) – генерация документации API.
8. Docker – контейнеризация приложения и зависимостей. Позволяет запускать приложение в изолированной среде.

Swagger документация:
Описание всех эндпоинтов. Для каждого эндпоинта:
summary (1 строка) и короткое description
tags
Параметры: path, query (в т.ч. page, size, sort)
RequestBody: схема DTO + example
Responses: как минимум 200/201, 400 (валидация), 401/403 (безопасность), 404, 409 (конфликты), 500 — со схемой ErrorResponse и примерами
Контент-тайпы: application/json, для загрузки — multipart/form-data
Примеры запросов/ответов.
